# FitStep 프로젝트 발표 대본 (1분)

---

안녕하세요, FitStep 프로젝트를 발표하겠습니다.

FitStep은 "기존 헬스 앱은 왜 내 헬스장 기구도, 내 컨디션도 모를까?" 라는 문제에서 출발했습니다.

동작 방식의 핵심은, 사용자가 루틴 추천을 요청하면 **RAG + Python 계산 + 사용자 입력** 세 가지를 조합해 GPT 프롬프트를 조립한다는 점입니다.

RAG는 두 종류입니다. 하나는 사용자가 등록한 헬스장 기구 목록을 ChromaDB에서 꺼내 "이 기구로만 운동 짜"라는 제약으로 주입합니다. 다른 하나는 국민체육공단 공공데이터 950건에서 연령·성별·BMI가 유사한 실제 사례를 벡터 검색해 추천 근거로 주입합니다.

Python 계산도 프롬프트에 직접 들어갑니다. 최근 2일 운동 기록을 분석해 쉬어야 할 부위를 자동으로 제외하고, 운동별 수행 이력을 보고 무게를 7.5% 늘리는 점진적 과부하를 계산해 넣습니다. GPT는 이 모든 정보를 받아 루틴 JSON만 생성합니다.

이 프로젝트는 CLI 봇에서 시작해 Streamlit 웹, FastAPI 백엔드 분리, SingleAgent, 마지막으로 근력·유산소·재활 전문가가 분리된 MultiAgent까지 단계적으로 진화했습니다.

이상입니다. 감사합니다.
