# FitStep 프로젝트 발표 대본 (1분)

---

안녕하세요, FitStep 프로젝트를 발표하겠습니다.

FitStep은 "기존 헬스 앱은 왜 내 헬스장 기구도, 내 컨디션도 모를까?" 라는 문제에서 출발했습니다.

동작 방식을 간단히 설명하면, 사용자가 루틴 추천을 요청하면 총 세 가지 정보가 GPT 프롬프트에 동시에 주입됩니다. 첫째, 사용자 헬스장 기구 정보입니다. 기구 목록을 ChromaDB에 벡터로 저장해두고, 루틴 생성 시 해당 사용자의 기구만 검색해 GPT에게 넘깁니다. GPT는 없는 기구는 자동으로 제외하고 루틴을 짭니다. 둘째, 국민체육공단 공공데이터 950건입니다. 사용자와 연령·성별·BMI가 유사한 실제 사례를 RAG로 검색해 추천 근거로 활용합니다. 셋째, 운동 이력 분석입니다. Python 코드가 운동별 마지막 수행일을 계산해 쿨다운 중인 부위는 제외하고, 기록이 쌓이면 무게를 7.5% 늘리는 점진적 과부하를 자동으로 제안합니다.

이 프로젝트는 CLI 봇에서 시작해 Streamlit 웹, FastAPI 백엔드 분리, SingleAgent, 마지막으로 근력·유산소·재활 전문가가 분리된 MultiAgent까지 단계적으로 진화했습니다.

핵심 설계 원칙은 "검증·계산은 Python이, 자연어 생성만 GPT가" 입니다. GPT는 절대 계산을 신뢰할 수 없기 때문입니다.

이상입니다. 감사합니다.
