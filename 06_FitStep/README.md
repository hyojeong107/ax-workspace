# 💪 FitStep — 나만의 AI 헬스 코치

FitStep은 Python CLI 기반의 개인 맞춤형 헬스 관리 앱입니다.  
사용자의 체력 수준과 운동 목표를 분석하고, OpenAI API를 통해 매일 새로운 운동 루틴을 추천합니다.  
운동을 완료할수록 기록이 쌓이고, 점진적 과부하(Progressive Overload) 원리에 따라 난이도가 자동으로 올라갑니다.  
**RAG 기술**을 활용해 내가 다니는 헬스장의 기구 환경을 AI가 완벽히 이해하고 루틴을 구성합니다.

## 🚀 설치 및 실행 방법

> **Windows 사용자**는 [WINDOWS_SETUP.md](WINDOWS_SETUP.md)를 참고하세요.

### 1. 저장소 클론

```bash
git clone <저장소_URL>
cd 06_FitStep
```

### 2. 가상환경 생성 및 활성화

```bash
# macOS / Linux
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate
```

### 3. 패키지 설치

```bash
pip install -r requirements.txt
```

### 4. 환경 변수 설정

`.env.example`을 복사해서 `.env` 파일을 만들고, 실제 값을 입력합니다.

```bash
cp .env.example .env
```

> 상세 항목은 아래 [환경 변수 설정](#환경-변수-설정) 섹션을 참고하세요.

### 5. MySQL 준비

MySQL이 실행 중인지 확인합니다. 데이터베이스와 테이블은 **앱 최초 실행 시 자동 생성**됩니다.

```bash
mysql -u root -p
```

### 6. 헬스장 기구 데이터 적재 (RAG)

샘플 데이터(`data/gym_1.json`)를 벡터 DB에 임베딩합니다.

```bash
python3 seed_gym.py
```

또는 앱 실행 후 메뉴 `4`에서 직접 입력해도 됩니다.

### 7. 앱 실행

```bash
python3 main.py
```

## 🗄 데이터베이스 구조

### MySQL — 관계형 DB

앱 실행 시 `fitstep` 데이터베이스 안에 아래 4개 테이블이 자동 생성됩니다.

#### `users` — 사용자 프로필

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INT (PK) | 자동 증가 고유 ID |
| name | VARCHAR(100) | 이름 |
| age | INT | 나이 |
| gender | VARCHAR(10) | 성별 (male / female / other) |
| height_cm | FLOAT | 키 (cm) |
| weight_kg | FLOAT | 몸무게 (kg) |
| fitness_level | VARCHAR(20) | 체력 수준 (beginner / intermediate / advanced) |
| goal | VARCHAR(200) | 운동 목표 (복수 선택 시 쉼표로 구분 저장) |
| health_notes | TEXT | 건강 주의사항 (부상 이력 등) |
| created_at | DATETIME | 등록 일시 |

#### `routines` — AI 추천 루틴

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INT (PK) | 자동 증가 고유 ID |
| user_id | INT (FK) | 사용자 ID |
| routine_date | DATE | 루틴 날짜 (하루 1개 생성) |
| exercises_json | TEXT | 운동 목록 JSON — `{ name, category, sets, reps, weight_kg, tip }` |
| ai_advice | TEXT | AI 맞춤 조언 메시지 |
| is_completed | TINYINT(1) | 완료 여부 (0: 미완료, 1: 완료) |
| created_at | DATETIME | 생성 일시 |

#### `workout_logs` — 운동 완료 기록

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INT (PK) | 자동 증가 고유 ID |
| user_id | INT (FK) | 사용자 ID |
| routine_id | INT (FK) | 어떤 루틴에서 수행했는지 |
| exercise_name | VARCHAR(100) | 운동 이름 |
| sets_done | INT | 실제 수행한 세트 수 |
| reps_done | INT | 실제 수행한 반복 수 |
| weight_kg | FLOAT | 사용한 무게 (맨몸이면 0) |
| note | TEXT | 사용자 메모 |
| logged_at | DATETIME | 기록 일시 |

#### `exercises` — 운동 종목 라이브러리 (향후 확장용)

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INT (PK) | 자동 증가 고유 ID |
| name | VARCHAR(100) | 운동 이름 (한국어) |
| name_en | VARCHAR(100) | 운동 이름 (영어) |
| category | VARCHAR(50) | 부위 (가슴·등·하체·어깨·팔·복근·유산소) |
| difficulty | VARCHAR(20) | 난이도 |
| equipment | VARCHAR(50) | 필요 기구 |
| description | TEXT | 운동 설명 |

### ChromaDB — 벡터 DB

헬스장 기구 정보를 임베딩해 저장합니다. `data/chroma_db/`에 로컬 파일로 저장됩니다.

| 컬렉션 | 문서 단위 | 메타데이터 |
|--------|----------|-----------|
| `gym_equipment` | 헬스장 요약 1개 + 기구별 1개씩 | `user_id`, `doc_type`, `eq_name` |

## 🧠 핵심 로직 설명

### 1. AI 운동 루틴 추천 흐름

```
사용자 프로필 (목표, 체력 수준, 몸무게, 주의사항)
        +
최근 7일 완료 운동 이력
        +
점진적 향상 분석 결과 (레벨업 여부, 다음 목표)
        +
[RAG] ChromaDB → 내 헬스장 기구 정보 검색
        ↓
OpenAI GPT-4o-mini 에 프롬프트 전송
  → 헬스장에 없는 기구 운동은 절대 추천 금지 지시
  → 무게는 사용자 몸무게 기준으로 직접 계산해 kg 숫자로 반환 요구
        ↓
JSON 형식으로 오늘의 루틴 수신
  { name, category, sets, reps, weight_kg(숫자), tip }
        ↓
MySQL routines 테이블에 저장
        ↓
Rich 라이브러리로 보기 좋게 출력
  → 세트 / 횟수 / 무게(kg) 컬럼 분리 표시
  → weight_kg == 0 이면 "맨몸" 으로 출력
```

### 2. 헬스장 기구 RAG 흐름

사용자가 다니는 헬스장의 기구 환경을 벡터 DB에 저장하고, 루틴 생성 시 검색해 AI 프롬프트에 주입합니다.

```
[등록 단계]
사용자가 기구 목록 입력 (메뉴 4 또는 seed_gym.py)
        ↓
OpenAI text-embedding-3-small 으로 임베딩
        ↓
ChromaDB에 user_id 메타데이터와 함께 저장
  - 문서 1: 헬스장 전체 요약 (이름 + 보유 기구 전체 목록)
  - 문서 2~N: 기구별 상세 (이름 / 수량 / 무게범위 / 특이사항)

[추천 단계]
루틴 생성 요청
        ↓
ChromaDB에서 해당 user_id 의 기구 정보 전체 검색
        ↓
AI 프롬프트에 주입:
  "스쿼트 렉 1개 (붐빌 때 사용 어려움), 덤벨 최대 40kg, 스미스 머신 1개..."
  "→ 위 목록에 없는 기구 운동은 절대 추천 금지"
        ↓
AI가 헬스장 환경에 완벽히 맞는 루틴 생성
```

**등록한 헬스장 기구를 수정하면** 오늘 루틴 캐시가 자동 초기화되어 즉시 새 루틴을 받을 수 있습니다.

### 3. 다중 목표 처리

운동 목표를 여러 개 선택할 수 있습니다 (예: `체중 감량, 근육 증가`).  
AI 프롬프트에 **모든 목표를 함께 전달**하여 각 목표에 맞는 운동이 균형 있게 섞인 루틴을 구성합니다.

- `체중 감량` 포함 시 → 유산소 운동 반드시 포함
- `근육 증가` 포함 시 → 고중량 복합 운동 위주
- `재활 / 부상 회복` 포함 시 → 건강 주의사항과 결합해 저강도 운동 선택

### 4. 점진적 향상 (Progressive Overload) 분석

운동 기록이 쌓일수록 자동으로 난이도가 올라가는 구조입니다.

**레벨업 판단 기준:**

```
아래 조건을 모두 만족하면 "레벨업 권장" 판정
  ① 해당 운동 기록이 2회 이상 존재
  ② 최근 기록의 세트 수 ≥ 3세트
  ③ 최근 기록들의 평균 반복 수 ≥ 11회
```

**레벨업 권장 시 제안:**

| 운동 종류 | 다음 단계 |
|-----------|-----------|
| 맨몸 운동 | 목표 횟수 +2회 또는 더 어려운 변형 동작 |
| 기구 운동 | 현재 무게 × 1.075 (약 7.5% 증가) |

이 분석 결과는 AI 프롬프트에 포함되어, AI가 `레벨업 권장` 운동을 더 어려운 버전으로 교체하거나 `weight_kg` 필드에 권장 수치를 반영합니다.

### 5. 파생 운동 추천

오늘 완료한 운동들을 분석해, 해당 운동과 연계되거나 발전된 운동을 다음 날 루틴에 자연스럽게 포함시킵니다.

```
예시:
  푸쉬업(무릎) → 일반 푸쉬업 → 인클라인 푸쉬업 → 딥스
  스쿼트       → 고블릿 스쿼트 → 바벨 백스쿼트
```

### 6. 이어하기 기능

운동 도중 앱을 종료해도 기록이 유지됩니다.  
다음에 `운동 완료 기록하기`를 선택하면, 이미 기록된 운동은 자동으로 건너뛰고 남은 운동부터 이어서 진행합니다.
