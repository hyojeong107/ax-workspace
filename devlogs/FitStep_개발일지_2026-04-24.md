### 3. 공공데이터 초기 인덱싱 스크립트 작성 (Phase 0 - 인덱싱 파이프라인 고도화)

**파일**: `08_FitStep_API/init_public_data.py`

**적용된 고도화 전략 4가지**

#### 전략 1. 문서 종류별 인덱싱 전략 분리
- 체력측정: 1레코드 → 자연어 문장 1개로 변환 후 1문서로 저장
  - 예) `"64세 여성, 신장 158.7cm, 체중 51.7kg, BMI 24.4, 윗몸일으키기 22회. 등급: 4등급."`
- 운동추천: (연령대+BMI등급+성별+상장등급+운동단계) 조합 → 추천운동 묶어서 1문서
  - 예) `"30대 여성 1단계비만 1등급 준비운동: 걷기, 스트레칭, 자전거타기"`
- 헬스장 기구: 기존 방식 유지 (요약 1개 + 기구별 개별)

#### 전략 2. 청킹 전 문서 구조 보존 전처리
- `MESURE_IEM_XXX_VALUE` 코드 → 한글 필드명 변환
  - 001=신장, 002=체중, 003=체지방률, 005=허리둘레, 006=수축기혈압, 007=BMI, 008=윗몸일으키기, 018=제자리멀리뛰기
- null 값 필드 문서에서 제외 (빈 정보로 임베딩 품질 저하 방지)
- 성별코드 변환: M→남성, F→여성
- CSV 따옴표/공백 strip 처리

#### 전략 3. 검색에 유리한 메타데이터 확장
- 체력측정 메타데이터: `source`, `age_group`, `gender`, `grade`, `bmi`(수치), `bmi_category`(저체중/정상/과체중/비만), `age`(정수)
- 운동추천 메타데이터: `source`, `age_group`, `gender`, `bmi_grade`, `award_grade`, `exercise_step`, `exercise_count`

#### 전략 4. 증분 인덱싱 지원
- `force_reindex=False` (기본): 컬렉션에 데이터 있으면 스킵
- `force_reindex=True`: 기존 삭제 후 재인덱싱
- 실행: `python init_public_data.py` / `python init_public_data.py --force`

**ChromaDB 컬렉션 구조 (최종)**
```
chroma_db/
├── gym_equipment          # 사용자별 헬스장 기구 (user_id 필터)
├── fitness_measurement    # 체력측정 공공데이터 950건
└── exercise_recommendation # 운동추천 공공데이터 (조합별 묶음)
```

### 5. Phase 2 - Contextual Retrieval 적용 (Contextual Embedding + Contextual BM25 + RRF Hybrid Search)

**파일**: `08_FitStep_API/init_public_data.py`, `08_FitStep_API/app/retrieval.py`, `08_FitStep_API/requirements.txt`

**배경**: Anthropic이 제안한 Contextual Retrieval 기법 적용. 단순 벡터 검색의 컨텍스트 부재 문제를 해결하고 키워드 검색을 병합해 검색 정확도 향상.

#### init_public_data.py 변경

- `_get_claude_client()`: `claude-haiku-4-5-20251001` 클라이언트 초기화
- `_generate_context(dataset_desc, chunk_text)`: 청크당 Claude 호출 → `"이 청크는 [내용 요약]."` 형식 컨텍스트 1문장 생성
- `_make_contextual_text()`: `"이 문서는 [데이터셋설명]. [컨텍스트]\n[원본청크]"` 형식으로 조합
- **Contextual Embedding**: 컨텍스트가 붙은 텍스트로 ChromaDB 임베딩 저장
- **Contextual BM25**: 동일 텍스트를 `rank_bm25.BM25Okapi`로 인덱싱 → `data/bm25_fitness.pkl`, `data/bm25_exercise.pkl`로 저장
- metadata에 `original_text` 필드 추가

#### retrieval.py 변경

- `_rrf_fusion(vector_ids, bm25_ids)`: RRF 공식 `score = 1/(60 + rank)` 적용, 두 결과 합산
- `retrieve_fitness_context()`: 벡터 검색 10개 + BM25 검색 10개 → RRF → 최종 상위 5개 반환
- `retrieve_exercise_recommendation()`: 동일 Hybrid Search 적용 (메타데이터 필터 우선, 히트 없으면 폴백)
- BM25 `.pkl` 파일 없으면 벡터 검색 단독으로 graceful fallback

#### requirements.txt 추가
```
anthropic>=0.40.0
rank_bm25>=0.2.2
```

**검색 흐름 요약**
```
쿼리
 ├─ 벡터 검색 (ChromaDB cosine) → 상위 10개
 ├─ BM25 검색 (rank_bm25)       → 상위 10개
 └─ RRF 융합 (k=60)             → 최종 상위 5개
```

### 7. Phase 3 - Reranking 적용 (FlashRank Cross-Encoder)

**파일**: `08_FitStep_API/app/retrieval.py`, `08_FitStep_API/requirements.txt`

**배경**: Bi-Encoder(임베딩) 기반 벡터 검색은 빠르지만 정밀도가 낮음. Cross-Encoder는 쿼리와 문서를 함께 보고 관련도를 재채점하므로 정확도가 높음.

**변경 내용**

- `retrieval.py`
  - 상수 변경: `FINAL_TOP_K = 5` → `RRF_TOP_K = 10`, `FINAL_TOP_K = 3` 분리
  - `_get_reranker()`: `ms-marco-MiniLM-L-12-v2` 모델 싱글턴 초기화
  - `_rerank(query, docs, top_k)`: FlashRank `RerankRequest`로 재채점 후 상위 k개 반환
  - `retrieve_fitness_context()`, `retrieve_exercise_recommendation()`: RRF 후보 10개 → `_rerank()` → 최종 3개 반환

- `requirements.txt`: `flashrank>=0.2.10` 추가

**검색 흐름 (Phase 3 완료 후)**
```
쿼리
 ├─ 벡터 검색 (ChromaDB cosine) → 상위 10개
 ├─ BM25 검색 (rank_bm25)       → 상위 10개
 ├─ RRF 융합 (k=60)             → 상위 10개 후보
 └─ FlashRank Reranking          → 최종 상위 3개
```

**설치**
```bash
venv\Scripts\pip install flashrank
```

### 11. Phase 4.5 - 운동 쿨다운 기반 다양성 순환 로직

**파일**: `07_FitStep_Web/app.py`

**배경**: 기존 루틴 추천은 최근 2일간 훈련한 부위를 제외(`avoid_section`)하는 것이 전부였음. 운동별로 마지막 수행일과 부위별 회복 기간을 반영해 오늘 추천 가능 여부를 GPT에 명시적으로 전달하는 로직이 없었음.

**추가 함수**

| 함수 | 역할 |
|---|---|
| `_get_exercise_cooldown_info(user_id)` | `/db/logs/progression` 재사용 → 운동별 마지막 수행일·경과일수·쿨다운 기준·가용 여부 반환 |
| `_build_cooldown_section(cooldown_info)` | 오늘 수행 / 쿨다운 중 / 재추천 가능 3분류로 GPT 프롬프트 섹션 생성 |

**쿨다운 기준 (부위별)**

| 부위 | 쿨다운 (일) |
|---|---|
| 가슴 / 등 / 어깨 / 하체 | 3일 |
| 팔 / 복근 | 2일 |
| 유산소 | 1일 |

**프롬프트 섹션 예시**
```
[운동 순환 스케줄 — 반드시 준수]
- 오늘 이미 수행한 운동 (오늘 루틴에서 제외): 런닝머신
- 쿨다운 중 — 오늘 제외: 스쿼트(2일 후 재추천), 바벨 벤치프레스(1일 후 재추천)
- 오랫동안 쉰 운동 (우선 추천): 랫풀다운, 레그프레스
- 재추천 가능한 운동: 플랭크, 덤벨 숄더프레스
※ 위 제외 운동 대신 같은 부위의 다른 변형 동작으로 대체해 다양성을 확보하세요.
```

**수정 내용**
- `_build_routine_prompt()`: `cooldown_info` 파라미터 추가, `cooldown_section` 삽입 위치 `safety_rule` 뒤
- `recommend_routine()`: `_get_exercise_cooldown_info(user["id"])` 호출 후 `_build_routine_prompt()`에 전달
- API 추가 없음 — 기존 `/db/logs/progression/{user_id}` 재사용
